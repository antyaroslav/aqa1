# Bug Report: Неверный расчет остатка при покупке на сумму порога кэшбэка

## Информация о баге
- **ID:** BUG-CASHBACK-001
- **Тип:** Логическая ошибка
- **Приоритет:** Средний
- **Статус:** Открыт

## Описание проблемы
При покупке товаров на сумму, равную порогу кэшбэка (1000 руб.), сервис возвращает значение 1000 вместо 0.

## Шаги воспроизведения
1. Создать объект класса `CashbackHackService`
2. Вызвать метод `remain(1000)`
3. Получить результат = 1000

## Фактический результат
Метод возвращает `1000` (требуется доплата)

## Ожидаемый результат
Метод должен вернуть `0` (порядок кэшбэка достигнут)

## Логи
Input: amount = 1000
Formula: 1000 - (1000 % 1000) = 1000 - 0 = 1000
Expected: 0
Actual: 1000
org.opentest4j.AssertionFailedError: expected:<0> but was:<1000>


## Причина ошибки
Формула не учитывает случай, когда `amount` делится на `boundary` без остатка.

## Рекомендованное исправление
```java
public int remain(int amount) {
    int remainder = amount % boundary;
    return remainder == 0 ? 0 : boundary - remainder;
}
